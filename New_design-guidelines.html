Design Guidelines v1.0

Wellness & Productivity App
Design System

A complete specification for building a dark-first, premium wellness Android app — covering colors, typography, components, motion, and all feature modules.

01 Design Principles
01
Dark-First, Always
Every screen is designed for dark mode as the primary experience. Light mode is optional. Surfaces use layered dark tones — never pure black — to create depth and hierarchy.
02
Orange as Signal
Warm orange is the sole accent color. It draws the eye to exactly one focal point per screen — active state, primary CTA, or key metric. No competing accent colors.
03
Calm Density
Information is organized in rounded, contained cards with generous internal padding. The layout breathes. No element touches another without explicit visual separation.
04
Touch-Native
Every interactive element is at minimum 48×48dp. Critical actions like toggles and FABs are thumb-reachable. Bottom-anchored navigation ensures one-handed operation.
05
Purposeful Motion
Animations serve orientation, not decoration. Screen transitions, card expansions, and toggle state changes all use physics-based easing. Duration is always under 350ms.
06
Feature Color Tokens
Each wellness module has its own accent token (mood = purple, water = teal, periods = pink). Accent only appears on that module's cards and charts — never globally.

07 Feature Module Specifications

Habit Tracker
Accent: #FF6B00. Show daily habits as a horizontal scrollable list of round icon cards. Each card shows: icon, habit name, streak count, and a circular progress ring. Completed state uses filled orange ring. Show weekly "heatmap" grid below (7 col × n rows, orange dot = completed).
Circular progress Streak counter Heatmap grid Swipe-to-complete

Mood Tracker
Accent: #9B59B6. Large emoji selector row (5 states: 😢😕😐🙂😄) with a tap-select animation. Below shows a 7-day mood trend line chart using purple gradient. Add an optional emotion tag row (stressed, grateful, tired…) using purple chips.
Emoji picker row 7-day trend chart Emotion tags Calendar heatmap

Journaling
Accent: #3498DB. Full-screen writing mode with dark background (#0D0D0F), minimal chrome. Show date + weather tag at top. Support rich text (bold, italic, lists). Show past entries as date-sorted cards with first 2 lines of preview. Include a "prompt of the day" banner at top.
Full-screen editor Daily prompt Entry cards Mood tag on entry

Expenses Tracker
Accent: #2ECC71. Top card shows total spent this month vs. budget as large numbers. Below: donut chart of categories (food, transport, health…) using green shades. Transaction list uses a row layout: category icon | name + date | amount (right-aligned in green). Add/edit via bottom sheet.
Donut chart Category icons Budget bar Monthly summary

Focus Timer
Accent: #F39C12. Full-screen timer with a large circular progress ring (amber). Center shows countdown in DM Mono font at 56sp. Supports Pomodoro (25/5 split). Below the ring: session count dots and current task label. Background dims to near-black during active session. Ambient sound picker row.
Large circular ring Pomodoro mode Session history Ambient sounds

Accent: #1ABC9C. Center card shows an animated wave/fill inside a circular or cup graphic that fills as intake increases. Below: quick-add buttons (150ml, 250ml, 500ml) as teal chips. Show daily total vs. goal in large mono numbers. Reminder settings shown at bottom of screen.
Animated fill Quick-add buttons Goal progress Reminders

Accent: #E91E8C. Full calendar grid view. Period days highlighted with pink fill, fertile window with pink tint, predicted days with dashed pink border. Top card shows cycle day number + phase name. Below calendar: symptom log chips (cramps, bloating, mood…) and cycle length stats.

09 Motion & Animation
Screen Transitions
Shared element transition where possible. Slide-up for new screens, slide-down for back navigation. No cross-fades — always directional.
duration: 280ms · easing: FastOutSlowIn
Card Tap Feedback
Scale down to 0.97 on press, back to 1.0 on release. Ripple effect uses orange at 8% opacity.
scale: 0.97 · duration: 100ms · release: 200ms
Toggle / Checkbox
Thumb slides with spring physics. Track color animates from neutral to orange. Checkmark draws with a path animation.
duration: 200ms · spring: stiffness 400, damping 30
Progress Rings & Bars
Animate from 0 to current value on screen entry. Use decelerating interpolator. Rings draw clockwise from top (12 o'clock).
duration: 600ms · easing: DecelerateInterpolator
Bottom Sheet Entry
Slides up from bottom with spring. Background scrim animates from 0 to 60% opacity simultaneously.
duration: 320ms · spring: stiffness 300, damping 28
Number Counter / Values
All numeric values (water intake, focus time, expense totals) animate with a rolling counter when updated.
duration: 400ms · easing: EaseOut
FAB Entry
FAB scales in from 0.5 to 1.0 with a slight overshoot. On press, rotates 45° to become a close icon when expanded.
entry: 240ms · rotate: 200ms
List Stagger
On screen load, list items animate in from bottom with a 50ms stagger between each item. Max 6 items stagger, rest appear instantly.
per-item delay: 50ms · translateY: 24dp→0 · duration: 260ms
Rule: All animations respect the system's "Reduce Motion" accessibility setting. When enabled, replace all motion with instant state changes or simple opacity fades (<150ms).
10 Accessibility Standards
✓
Touch Targets minimum 48×48dp
All tappable elements including icons, nav items, toggles, and checkboxes must meet this minimum.
✓
Color Contrast ≥ 4.5:1
Primary text (#F0F0F2) on card background (#1E1E21) achieves 12.4:1. Secondary text (#A0A0A8) achieves 5.8:1. Always verify with a contrast checker.
✓
ContentDescription on all icons
Every icon button and decorative icon must have a meaningful contentDescription for TalkBack screen reader users.
✓
Reduce Motion support
Check AnimatorDurationScale. Replace all slide/scale animations with simple alpha fades when system preference is set.
✓
Never rely on color alone
Pair color with icons, patterns, or text labels. E.g., expense categories use both color and icon. Period calendar uses both pink fills and text labels.
✓
Scalable text (sp units only)
All font sizes must use sp (scalable pixels), never dp, to respect system font size settings.
