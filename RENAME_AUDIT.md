## Part 5 â€” Keystore

The keystore file itself does not need to be renamed â€” it's a binary file referenced by path in the build config. However:

1. **Rename the file** `android/app/habitgenius-release.keystore` â†’  
   `android/app/<newname>-release.keystore`
2. Update all references to the filename in:
   - `android.yml` (line 131)
   - `codemagic.yaml` (line 157)
   - `firebase-sha-fingerprints.md` (line 51)
3. **Keystore alias** (`habitgenius`) is embedded in the keystore and cannot be changed without creating a new keystore. If you want the alias to match the new name, generate a new keystore with `keytool`.
4. Update `CM_KEY_ALIAS` secret in GitHub / Codemagic to match.

## Part 6 â€” Firebase Console (manual steps)

> These cannot be done by editing code â€” they require the Firebase Console UI.

1. Go to [Firebase Console](https://console.firebase.google.com) â†’ project **habitgenius-ec640**
2. **Project settings â†’ Your apps â†’ Android app**
3. Add a new Android app with the new package name
4. Download the new `google-services.json` and replace `android/app/google-services.json`
5. Re-register all SHA-1 / SHA-256 fingerprints for the new app entry
6. Update Firestore Security Rules if they reference `com.habitgenius`
7. If renaming the Firebase project itself: all Firestore/Auth data is preserved, but the project ID and storage bucket URL change

## Part 7 â€” Google Play Console (manual steps)

> Once an app is published under a package name, **that package name can never be changed**.  
> If the app has **not yet been published**, you can rename freely.

1. Create (or update) the app in Play Console with the new package name
2. Create a new in-app product with the new product ID (e.g. `<newname>_pro_lifetime`)
3. Update `kProProductId` in `purchase_service.dart` to match
4. Update `PACKAGE_NAME` in `codemagic.yaml` and `android.yml`

## Risk Notes

| Risk | Detail |
|---|---|
| **Existing user data loss** | If `_kFileName` is changed, the app won't find users' existing `habitgenius_data.json`. Add a migration in `DataService.resolveFilePath()` to copy/rename the old file. |
| **Published app package lock** | Once published on Play Store, `applicationId` (`com.habitgenius`) is permanently locked to that listing. Changing it means creating a new Play Store listing and losing reviews, installs and rankings. |
| **Firebase SHA fingerprints** | After adding the new Android app to Firebase, re-add all SHA-1/SHA-256 fingerprints (debug + release). Google Sign-In will fail until this is done. |
| **Keystore alias** | The alias is baked into the binary keystore file. If a new keystore is created for the new alias, you must use it consistently â€” mixing old/new keystores will cause signature mismatch errors on the Play Store. |
| **In-app product ID** | `kProProductId` must exactly match the product created in Play Console. A mismatch means `queryProductDetails` returns empty and the upgrade button shows no price. |
